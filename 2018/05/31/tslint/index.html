tslint.json 设置

tslint主要关注三部分：extends，rulesDirectory，rules我们分别来看一下

extends

表示从内建的预设配置继承，可以是单一也可以是数组，内建的预设配置如下

tslint:recommended:最常用，最稳定的一些规则选项，

tslint:latest:继承自tslint:recommended，根据最新的一些特性进行规则设置，可能是下一个版本的tslint:recommended

tslint:all: 打开严格模式的所有规则选项

预设也可以使用第三方的，比如tslint-config-alloy，需要安装

rulesDirectory

指定规则生效的目录，可以是一个或多个

rules

单独定义的一些规则选项，会覆盖之前的一些预设的选项，示例如下

注释标记

  • /* tslint:disable */ - 忽略该行以下所有代码出现的错误提示
  • /* tslint:enable */ - 对该行以下所有代码启用规则
  • /* tslint:disable:rule1 rule2 rule3… */ - 忽略该行以下所有代码出现的错误提示（指定忽略规则）
  • /* tslint:enable:rule1 rule2 rule3… */ - 对该行以下所有代码启用规则（启用规则）
