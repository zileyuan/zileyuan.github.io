建议使用简化版

简化版各种标识

  • ??表示未跟踪，
  • D表示删除
  • A表示新增
  • M表示修改
  • R表示改名
  • UU表示冲突（不会存在于暂存区，解决后显示M）
  • 蓝色表示已经在暂存区，红色表示未在暂存区

修改提交

git add

  • add意义是把变化加入暂存区，而不仅仅是新指文件

    • 跟踪新文件，并放入暂存区（?? -> A）
    • 已经跟踪过的文件（修改或删除），则放入暂存区（仅仅颜色变化)
    • 标记解决冲突的文件（UU -> M）

  • 命令演示：

git mv

  • 改名并直接进入暂存区

git rm

  • 删除并直接进入暂存区

  • 停止跟踪文件但不删除文件

git diff

  • 比较版本文件

撤销操作

基本概念

HEAD

  • 在Git中，用HEAD表示当前版本，它是一个指针，指向当前所在的本地分支（将 HEAD 想象为当前分支的别名），也就是最新的提交

  • 上一个版本就是HEAD^，上上一个版本就是HEAD^^，当然往上n个版本写n个^比较容易数不过来，所以写成HEAD~n，例如： HEAD~50。

  • 在命令中commit参数都可以用相关HEAD填充。

重新提交

  • 说明：有时候我们提交完了才发现漏掉了几个文件没有添加，或者提交信息写错了。此时，可以尝试重新提交

  • 用法

  • 举例

git checkout

用法：commit之前撤销指定的文件修改

注意：git checkout – file命令中的–很重要，没有–，就变成了“切换到另一个分支”的命令，后面的分支操作中会再次遇到git checkout命令

git reset

当前版本回退到之前commit的任何版本

git revert

撤销一个已经提交的快照，通过在最后加上一个撤销了更改的新提交，而不是从项目历史中移除这个提交。这避免了Git丢失项目历史，这一点对于你的版本历史和协作的可靠性来说是很重要的。

远程仓库

基本概念

  • origin
    “origin” 是当你运行 git clone 时默认的跟踪远程仓库名字。如果你运行 git clone -o foo，那么你默认的跟踪远程分支名字将会是 foo/master。
  • 跟踪
    远程跟踪分支是远程分支状态的引用（别名）。举例来说：假设你的网络里有一个在 git.ourcompany.com 的 Git 服务器。如果你从这里克隆，Git 的 clone 命令会为你自动将其命名为 origin，拉取它的所有数据，创建一个指向它的 master 分支的指针，并且在本地将其命名为 origin/master。Git 也会给你一个与 origin 的 master 分支在指向同一个地方的本地 master 分支，这样你就有工作的基础。（我们在讲解fetch和pull就会比较好理解了）
